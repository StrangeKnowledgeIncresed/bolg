单例 Singleton

1.饿汉式

类加载内存后，就实例化一个实例，jvm保证线程安全(jvm保证每个class只加载一次，静态变量只要load到内存后就会初始化）

唯一缺点：类加载时就实例化，不管用到与否

public class Singleton01 {

    private static final Singleton01 INSTANCE = new Singleton01();

    private Singleton01() {
    }

    public static Singleton01 getINSTANCE() {
        return INSTANCE;
    }
}

2.懒汉式（懒加载）

加载的时候不new 对象，在要使用的时候才初始化化实例

缺点：线程不安全

public class Singleton02 {

    private static  Singleton02 instance;

    public static Singleton02 getInstance() {
        if(instance==null){

            instance=new Singleton02();
        }
        return instance;
    }
}

懒汉式（加锁）

缺点：效率下降（每次进行加锁操作）

public class synchronized Singleton02 {

    private static  Singleton02 instance;

    public static Singleton02 getInstance() {
        if(instance==null){

            instance=new Singleton02();
        }
        return instance;
    }
}

双重校验

public class Singleton03 {
    public static volatile Singleton03 instane;


    public static Singleton03 getInstane() {


        if (instane == null) {
            synchronized (Singleton03.class) {
                if (instane == null) {
                    instane = new Singleton03();

                }
            }
        }


        return instane;
    }
}

静态内部类

jvm保证单例，线程安全

静态内部类不会随着类的加载而加载，只有当getInstance时候才会去加载静态内部类的实例

public class Singleton04 {
    private Singleton04() {
    }


    private static class Singleton04Holder {

        private static final Singleton04 instance = new Singleton04();
    }


    public Singleton04 getInstance() {

        return Singleton04Holder.instance;
    }
}

策略 Strategy

对于实现同一个目的而产生的不同的策略

相当于switch 和if else的封装

经典用例：java比较器Comparator

实现Comparator 重写compare方法 可以对不同的对象实现不同的比较策略

public class person {
    int age;

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}

public class stu extends person implements Comparator<person> {

    @Override
    public int compare(person o1, person o2) {
        if (o1.age > o2.age) {
            return 1;
        } else if (o1.age < o2.age) {
            return -1;
        } else {
            return 0;
        }
    }
}

对于对象数组的排序实现不同的比较策略 (new一个自定义的Comparator)

  public static void main(String[] args) {
  //对数组a进行排序可以传入不同的比较策略
        stu[]a={new stu()};
        Arrays.sort(a, (o1, o2) -> o1.getAge()==o2.getAge()?0:o1.getAge()>o2.getAge()?1:-1);

工厂Factory

主要用于对象的创建

简单工厂

public class SingleFactory {


    public Bread createBread() {
        return new Bread();
    }

    public static void main(String[] args) {
        SingleFactory singleFactory =new SingleFactory();
        Bread bread = singleFactory.createBread();
    }
}

静态工厂

public class StaticFactory {


    public static  Bread createBread() {
        return new Bread();
    }

    public static void main(String[] args) {
        Bread bread = StaticFactory.createBread();
    }

抽象工厂Factory

(抽象产品组 实现不同产品组的量产)

例如：一组生产白人和奔驰的工厂，一组生产黑人和宝马的工厂

public abstract class AbstractFractory {

    abstract Car createCar();

    abstract Person createPreson();
}


public abstract class Person {
}
public abstract class Car {
}
public class BlackPerson extends Person{
}
public class WhitePerson extends Person{
}
public class  Benz extends Car{
}
public class Bmw extends Car{
}


public class BlackPersonAndBmwFactory extends AbstractFractory {
    @Override
    Car createCar() {
        return new Bmw();
    }

    @Override
    Person createPreson() {
        return new BlackPerson();
    }
}
public class WhitePersonAndBenzFactory extends AbstractFractory {
    @Override
    Car createCar() {
        return new Benz();
    }

    @Override
    Person createPreson() {
        return new WhitePerson();
    }
}

装饰器Decorator

在原有功能上的扩展

例：功能为打包，扩展为将钱和杯子装进书包

抽象构件：Pack

装饰者：Decortor

被装饰者：concertPack

装饰：Money,Cap

public abstract class Pack {

    public abstract void toPack();
}

public class Decorator extends Pack {
    public Pack p;

    public void setP(Pack p) {
        this.p = p;
    }
    @Override
    public void toPack() {
        if (p != null) {
            p.toPack();
        }
    }
}

public class ConcertPack extends Pack {
    @Override
    public void toPack() {
        System.out.println("打开书包");
    }
}

public class Money extends Decorator{
    @Override
    public void toPack() {
        super.toPack();
        System.out.println("将钱放入书包");
    }
}

public class  Cap extends Decorator{
    @Override
    public void toPack() {
        super.toPack();
        System.out.println("将杯子放入书包");
    }
}

调用

 public static void main(String[] args) {
        ConcertPack concertPack =new ConcertPack() ;
        CapPack capPack =new CapPack() ;
        MoneyPack moneyPack =new MoneyPack();
        capPack.setP(concertPack);
        moneyPack.setP(capPack);
        moneyPack.toPack();

    }

结果

打开书包
将杯子放入书包
将钱放入书包

这样就可以通过装饰器动态扩展装饰者和被装饰者，并且不用去修改其他的类
